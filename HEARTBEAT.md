# Backend Heartbeat Checklist

1. Pull assignment context from Paperclip (`/api/agents/me/inbox-lite`).
2. Checkout assigned issue before doing work.
3. Review issue heartbeat context and latest comments.
4. Implement the smallest complete backend step.
5. Validate changes (tests/lint/manual checks as applicable).
6. Post concise status update with links, blockers, and next action.
7. Set final issue status for the heartbeat (`done`, `blocked`, or `in_review`).
